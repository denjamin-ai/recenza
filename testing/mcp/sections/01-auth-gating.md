- ✅ 1. GET `/reviewer` гостем → редирект на `/login` (БЕЗ `?next=`).
- ✅ 1. GET `/admin/dashboard` гостем → редирект на `/admin/login` (БЕЗ `?next=`).
- ✅ 2. Логин reader через UI → редирект на `/`; меню пользователя зафиксировано; «Выйти» работает (остаёмся на `/`, снова видна «Войти»).
- ✅ 3. Неверный пароль → `role=alert` с текстом «Неверный никнейм или пароль.», остаёмся на `/login`, POST `/api/auth/user` → 401.
- ✅ 4. Логин author → редирект на `/author` (title «Кабинет автора | Recenza»). Logout OK (с `/author` после «Выйти» уводит на `/`).
- ✅ 4. Логин reviewer → редирект на `/reviewer` (⚠️ title остаётся общий «Recenza — девблог с редакционным ревью», не «Кабинет ревьюера»). Logout OK.
- ✅ 5. Интент-реплей: гость кликает «Полезно» на `/blog/async-deep-dive/event-loop` → редирект на `/login?next=%2Fblog%2Fasync-deep-dive%2Fevent-loop&intent=vote%3Achp_published%3A1`; после логина reader голос реплеится (POST `/api/chapters/chp_published/vote` → 200) и возвращает **на главу** (`/blog/async-deep-dive/event-loop`).
  ⚠️ Нюанс: реплей — это toggle; у reader в seed уже был апвоут (`chv_1`) → реплей его СНЯЛ (счётчик 2 → 1, `aria-pressed=false`). См. «Баги/аномалии».
- ✅ 5. Закладка гостем → `/login?next=%2Fblog%2Fasync-deep-dive%2Fevent-loop&intent=bookmark%3Ablog_async` (закладка — per-blog, id блога). Реплей не выполнялся (снял бы seed-закладку reader).
- ✅ 6. `/admin/login`: одно поле «Пароль» + кнопка «Войти как администратор». Вход `<ADMIN_PASSWORD_PLAIN из .env.test>` → `/admin/dashboard`. Полноэкранный портал (шапки сайта НЕТ), sidebar «Навигация админ-портала». Выход — кнопка **«Выйти к блогу»** внизу sidebar → уводит на `/` и УНИЧТОЖАЕТ админ-сессию (повторный `/admin/dashboard` → `/admin/login`).
- ✅ 7. Под reader: `/author` → редирект на `/`; `/admin/dashboard` → редирект на `/`; `/reviewer` → редирект на `/` (без `?next=`, без сообщения об ошибке).
- ✅ (бонус) Негативные статусы API: POST `/api/auth/user` wrong pwd → 401; POST `/api/auth` wrong pwd → 401; GET `/api/auth/user` гостем → 200 `{"user":null}` (но при «протухшей» сессии может отдавать 401 — видели в консоли).
- ✅ (бонус) `/login` при УЖЕ залогиненном пользователе НЕ редиректит — снова показывает форму (можно перелогиниться).

## Тайминги

- Dev-сервер :3001 был уже прогрет (роуты скомпилированы ранее) — компиляционных задержек в этой сессии не наблюдалось. ⚠️ После холодного старта `npm run dev:test` первый хит каждого роута компилируется секунды — в спеках закладывать увеличенный таймаут первой навигации.
- Warm-замеры (fetch, cache: no-store): `/` ≈ 70мс · `/login` ≈ 53мс · `/blog/async-deep-dive/event-loop` ≈ 145мс · `/admin/login` ≈ 41мс.
- Логин (POST + redirect + рендер) — визуально < 1с. Все навигации в сессии < 2с.

## Поведение и гочи

- **Редиректы гостя:** `/author` и `/reviewer` → `/login` **без** `?next=` (после логина интент-возврата на исходный protected-роут НЕТ; author попадёт в `/author` только потому, что это его дефолт). `/admin/dashboard` → `/admin/login`.
- **Дефолтные редиректы после логина:** reader → `/`; author → `/author`; reviewer → `/reviewer`. При наличии `?next=` в URL логина — уводит на `next`.
- **Protected под чужой ролью:** reader на `/author`, `/reviewer`, `/admin/dashboard` → молча редирект на `/` (без тоста/сообщения). Под **админ-сессией** `/author` → редирект на `/admin/dashboard` (наблюдалось со стейл-cookie в начале сессии).
- **Intent-replay:** формат `?next=<path>&intent=vote:<chapterId>:<1|-1>` / `intent=bookmark:<blogId>`. После логина действие выполняется автоматически (POST соответствующего API), затем переход на `next` — т.е. **возврат на главу**, не на `/`. Ссылка «Войдите» в комментариях несёт `next=<путь>#comments` (без intent).
- **Intent = toggle:** если действие уже было сделано ранее (seed-голос reader), реплей его СНИМАЕТ. Для тестов: после `test:reset` реплей `vote:chp_published:1` под reader уменьшает счётчик 2 → 1.
- **Logout:** пункт меню «Выйти»; со страницы `/` остаёмся на `/`; с protected (`/author`, `/reviewer`) — уводит на `/`. Admin-выход называется «Выйти к блогу» (кнопка в sidebar), уводит на `/` и рушит сессию.
- **Сессии взаимоисключающие:** admin-сессия ≠ user-сессия (`GET /api/auth/user` под админом → `{"user":null}`). Вход обычным пользователем при живой admin-cookie перезаписывает сессию (инвариант isAdmin XOR userId).
- **Rate-limit логина считает только ПРОВАЛЫ** (`src/lib/rate-limit.ts`): 5 неуспешных / 15 мин → 6-я попытка 429; успешный вход сбрасывает счётчик. Многократные успешные логины в тестах безопасны.
- `/login` и `/admin/login` — «голые» страницы без шапки сайта (только ссылка-логотип «Recenza» + заголовок).
- Админ-портал полноэкранный: шапки сайта нет, skip-link ведёт на `#admin-main`, свой header «Платформа · Сводка» + searchbox.
- Кнопки реакций у гостя выглядят обычными (не disabled) — клик уводит на логин с intent.
- Карусель промо авторотируется (~5 сек) — снапшоты/скриншоты ленты нестабильны; при кликах по шапке рядом с каруселью возможны перерисовки (первый клик по «Меню пользователя» совпал с ротацией — меню пришлось открывать повторно).

## Ответы на вопросы

- **Какой целевой URL после intent-replay?** Возврат **на главу**: `/blog/async-deep-dive/event-loop` (значение `next` из URL логина). Не `/`. Голос при этом уже применён (toggle) — отдельного тоста не замечено, состояние кнопок обновлено.

## Баги/аномалии

1. **[Поведение/спорно] Intent-replay может СНЯТЬ уже существующий голос/закладку.** Реплей выполняет тот же toggle-эндпоинт: гость кликнул «Полезно» → залогинился как пользователь, у которого голос уже стоял (seed `chv_1`) → голос снялся (2 → 1). Ожидание пользователя — «поставить голос», а не «переключить». То же относится к `bookmark:blog_async`. Рекомендация: реплеить идемпотентно (ставить, если не стоит) или не реплеить при совпадении состояния.
2. **[Минор] Гостевые редиректы protected-роутов теряют исходный URL:** `/author`, `/reviewer` → `/login` без `?next=` — после логина под нужной ролью на исходный роут не возвращает (роль-дефолт совпадает для «родных» ролей, но для диплинков вида `/author/blog/...` возврат потеряется — не проверялось глубже).
3. **[Минор] Title не обновляется:** `/reviewer` и весь админ-портал (в т.ч. `/admin/dashboard`) держат общий title «Recenza — девблог с редакционным ревью», тогда как `/author` имеет «Кабинет автора | Recenza». Непоследовательно; для тестов title как маркер годится только у author.
4. **[Данные seed] 404 на `/uploads/articles/event-loop.png`** (обложка/иллюстрация главы event-loop) — файла нет в `public/uploads` (загрузка — Фаза 12). Консоль ридера всегда содержит этот error → в console-мониторинге тестов заложить allowlist.
5. Прочие консоль-события: ожидаемые 401 от негативных проб `/api/auth*`; dev-warning о preload CSS (`link preload not used`) на `/login` — шум turbopack, не для проверок.
