Подождите секунду."}` + `Retry-After: 1`.

- ✅ Якорь-механизм end-to-end: выделение → плавающая кнопка «Комментировать» → чип «К фрагменту: «…»» → отправка → у комментария появляется «↑ «…»» / «Перейти к фрагменту в тексте».

## Тайминги

- `npm run test:reset` ≈ 5–8 с (migrate+seed).
- Dev-сервер :3001 был прогрет: первая загрузка `/`, ридера, `/login`, `/bookmarks`, `/u/author` ≈ 1–2 с; повторные < 0.5 с. На холодном сервере закладывать до нескольких секунд на первый хит роута (dev-компиляция).
- Комментарий/правка/удаление отражаются без перезагрузки за ~0.3–0.7 с (RSC refresh).
- Поллинг `/api/notifications` каждые 30 с + on focus.
- ⏱ Окно правки: cmt_fresh создан за 5 мин до seed — «Изменить» у него живёт лишь ~10 мин после `test:reset` (наблюдал исчезновение на 18-й минуте). Тесты COM-EDIT-WINDOW на seed-данных гонять сразу после ресета либо создавать свой комментарий.

## Поведение и гочи

- Мутации комментариев/голосов: rate-limit **1/сек на пользователя** → 429 «Слишком часто. Подождите секунду.» + `Retry-After: 1`. В спеках между мутациями одной сессии ставить ≥1.1 с паузы (или ждать ответа) — иначе ложные 429 (поймал 429 на PATCH сразу после POST).
- Статусы API (fetch same-origin): POST comment гость 401 «Требуется вход.»; troll 403 «Комментирование ограничено.»; reviewer 403 «Ревьюеры не участвуют в публичных обсуждениях.»; глубина>2 → 409 «Слишком глубокая вложенность.»; PATCH вне окна 403 «Окно редактирования истекло.»; PATCH чужого 403 «Нельзя редактировать чужой комментарий.»; PUT 405; голос за своё 403 «Нельзя голосовать за свой комментарий.»; повтор <1с 429.
- Удаление комментария — **native confirm** «Удалить комментарий?» (в Playwright — `page.on('dialog')`).
- Tombstone «Комментарий удалён» рендерится только при живых потомках; удалённый лист (и ветка, где все удалены) скрываются целиком; счётчик N их не считает.
- «Отправить» в композере disabled при пустом тексте — негатив «пустой POST» проверяется только через fetch (400 «Пустой комментарий.»).
- Подписка на автора недоступна с профиля `/u/{handle}` — только из ридера.
- Клик по уведомлению навигирует только если в payload есть `href` (или type=new_chapter): seed-уведомление «Ответ на ваш комментарий» (payload только `commentId`) при клике лишь помечается прочитанным, меню не закрывается.
- Гость: клики по реакциям ведут на `/login?next=…&intent=…` (реплей действия после входа).
- Плавающая кнопка якоря скрывается на scroll/resize/Escape — в тестах выделять текст мышью и не скроллить до клика (smooth-scroll между выделением и mouseup убивает кнопку — источник flaky).
- `time` в комментариях относительный («2 дн назад», «только что»), в ленте — абсолютный («17 июня 2026 г.») — для asserts не полагаться на точные строки времени.
- Логин-редиректы по ролям: reader → `/`, reviewer → `/reviewer` (учитывать в fixture-логине).

## Ответы на вопросы

**Точный анкор-механизм.** Комментарий привязывается выделением текста в прозе главы: на `mouseup`/`keyup` с непустым выделением появляется плавающая кнопка «Комментировать» (`aria-label="Прокомментировать выделенный фрагмент"`, `position:fixed` над выделением). Клик резолвит выделение в ближайший к началу выделения элемент `[data-block-id]` и шлёт `CustomEvent("recenza:comment-anchor")` с `{ chapterSlug, blockId, quote }`, где `quote` — текст выделения, усечённый до **600 символов**. В композере появляется чип «К фрагменту: «quote»» с кнопкой ✕ «Убрать привязку к фрагменту». В `POST /api/comments` уходит `anchor: { blockId, quote }` — **только у root-комментариев** (у ответов anchor принудительно null). Отдельной кнопки на блоке нет — только выделение. Отображение якоря в комментарии — кнопка «Перейти к фрагменту в тексте» с текстом `↑ «quote»`; клик скроллит к `[data-block-id]` (не к `id="block-…"`). Пример из seed: `{"blockId":"blk_el_p_1","quote":"неблокирующие операции"}`.

## Баги/аномалии

1. **404 на seed-ассеты** (console error при каждой загрузке): `/uploads/articles/event-loop.png` (image-блок главы — рендерится пустая рамка с alt «Схема цикла событий») и `/uploads/donations/sbp-qr.png` (QR в DonateModal — битая картинка). Seed ссылается на файлы, которых нет в `public/uploads`. Для smoke «нет console errors» придётся вайтлистить или добавить файлы.
2. **Уведомление comment_reply некликабельно**: seed-payload содержит только `commentId`, а `NotificationBell.hrefOf()` требует `payload.href` → клик не навигирует и не закрывает меню. Либо баг генератора уведомлений (не кладёт href), либо seed неполон.
3. **Интермиттентные «мёртвые» trusted-клики** (окружение/MCP, не аппликация): дважды подряд клик Playwright по «Полезно» и трижды по «Войти» не породил ни одного сетевого запроса и не изменил React-state, при этом `element.click()` через evaluate срабатывал мгновенно, а повторные trusted-клики позже — тоже. Вероятный фактор — dev-оверлей Next DevTools/hit-testing. Для автотестов: после клика ждать сетевой ответ или смену `aria-pressed`/URL и ретраить.
4. **Дублирующийся `nav[aria-label="Навигация по блогу"]`** (mobile `<details>` + desktop `<aside>`) — strict mode violation при поиске по aria-label; в спеках скоупить селектор.
5. Минорно: у textarea режима правки комментария нет accessible name (искать по `li#comment-{id} textarea`); tombstone из seed (`cmt_deleted`) невидим — для теста tombstone нужно создавать ветку с ответом.
