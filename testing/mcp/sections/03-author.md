## Тайминги

- **Гидрация /login после `test:reset` — главная ловушка.** Страница и форма рендерятся быстро, но submit-обработчик не навешан: первый клик по «Войти» и Enter НЕ дают POST /api/auth/user (форма молча «съедает» клики ~1–2 мин при холодном dev-сервере, пока компилируются клиентские чанки). Рабочая стратегия для спеков: после клика ждать `waitForURL('/author')` с ретраем клика, либо дождаться сетевого POST.
- Первый хит каждого роут-сегмента на dev-сервере компилируется секунды (2–10 с): /author, /author/blog/[slug], edit, review, /u/[slug]. Повторные загрузки — ~1 с.
- Сохранение редактора/портфолио — мгновенно (<0,5 с), индикатор меняется синхронно с ответом PATCH/PUT.
- Комментарий появляется без перезагрузки за ~1–2 с после POST (router.refresh).

## Поведение и гочи

- Логин `author` редиректит сразу на `/author` (не на ленту).
- Кнопка создания блога — плитка `button` «Новый блог» (текст в снапшоте «+ Новый блог»), НЕ ссылка.
- В footer-агрегате карточки блога «нужны правки» не выводится: только «N глав · N опубл.» и «N на ревью · N черн.» (changes-requested виден точками статусов и табом «Нужны правки (1)» в детали блога).
- Ссылка «← Кабинет» в редакторе ведёт на деталь БЛОГА (/author/blog/[slug]), не в /author.
- Редактор: заголовки (title и h2/h3-блоки) — INPUT, остальные блоки — TEXTAREA; accessible name текстбокса = тип блока («Параграф», «Заголовок 2»).
- «+ Блок» открывает `menu`/`menuitem`; слэш «/» в пустом параграфе — `listbox «Вставить блок»`/`option`. Набор пунктов идентичен (11). Escape закрывает слэш-меню.
- Сохранения по кнопке (автосейва нет), тостов нет — индикатор «нет изменений → не сохранено → сохранено». Dirty-флаг не сбрасывается при возврате значения к исходному (портфолио: toggle туда-обратно оставляет «не сохранено»).
- Изменение навыков внутри SubmitSheet не делает редактор dirty (стейт шита отдельный).
- SubmitSheet: match% пересчитывается вживую при правке навыков; «Топ» — это ПОРЯДОК сортировки списка (композит: навыки 50% + рейтинг 30% + объём 20%, `src/lib/reviewer-match.ts`), отдельного бейджа «Топ» в UI нет. Показываются: процент (например «100%»), занятость («свободен»/«занят»/«загружен»), компетенции чипами, совпавшие — с «✓».
- «загружен» (full) ⇒ чекбокс ревьюера disabled — выбрать нельзя.
- Блокировка submit — disabled-кнопка «Отправить» + подпись «Закройте все пункты» в футере шита (без тостов/алертов).
- Заблокированный редактор (under-review/published) — не 409-страница, а read-only вид: инфо-баннер, title readOnly, ни блоков, ни кнопок Сохранить/Отправить/⚙; остаётся ссылка «Просмотр».
- API: PATCH /api/author/chapters/chp_under_review → **409** `{"error":"Главу нельзя редактировать в текущем статусе."}`; chp_published → тот же **409**; чужая chp_ghost → **404** `{"error":"Глава не найдена."}`.
- **Rate-limit мутаций ~1/сек**: два PATCH подряд → **429** `{"error":"Слишком часто. Подождите секунду."}` — в API-тестах ставить паузы ≥1 с между мутациями.
- Портфолио: PUT /api/author/portfolio; тумблер «Видно всем» [pressed] ↔ «Скрыто».
- /u/author гостю отдаёт портфолио (fetch без cookie: 200, текст есть), «Редактировать»/«видно всем» — только владельцу.
- Review-POV автора: вердикт-кнопок нет (только у ревьюера); авторские действия — «Отметить решённым», «Применить и закрыть», «Сменить ведущего», «Отправить v2», чат.
- Комментарии: форма доступна автору в своём блоге; бейдж «автор» на его комментариях; кнопка «Отправить» disabled при пустом поле.
- Оценка ревьюеров: POST /api/author/ratings {chapterId, reviewerHandle, stars}; карточка в кабинете видна только пока есть неоценённый ревьюер опубликованной главы — в seed оба оценены (5★/4★), секции нет. Для e2e-теста оценки нужен сценарий с новой публикацией (кластер 4) или правка seed.

## Ответы на вопросы

- **Чужая глава (PATCH /api/author/chapters/chp_ghost): 403 или 404?** → **404** («Глава не найдена.») — ownership маскируется под отсутствие ресурса.
- **Кнопка «Опубликовать», когда не все approve: disabled или отсутствует?** → **Отсутствует в DOM.** В `action-bar.tsx` рендер условный: `{allApproved && active && <button>Опубликовать…}`. На promises (1 request-changes) её нет — вместо неё «Отправить v2» и статус «есть запрос правок». `disabled` у неё бывает только на время busy после клика.
- **Где карточка оценки 1–5★?** → Кабинет `/author`, секция `h2 «Оцените ревьюеров»` (карточка главы со звёздами по каждому ревьюеру; появляется после публикации, пока есть неоценённые).

## Баги/аномалии

1. **[minor/UX] Markdown-шорткат «## » теряет фокус и последующий ввод.** В пустом параграфе набор «## Текст» конвертирует блок в «Заголовок 2» сразу после «## », но фокус НЕ переносится в новый INPUT (элемент ремаунтится textarea→input), и весь текст, набранный после пробела, уходит в никуда (activeElement = body). Значение нового заголовка остаётся пустым. Код: `applyShortcut` в `src/app/author/_components/editor/markdown.ts` вызывается из `block-list-editor.tsx:82` без переноса фокуса. Воспроизводится и живой печатью.
2. **[info] Слэш-меню содержит 11 пунктов**, а не «14 типов» из CLAUDE.md/спеки (вариации list bullet/numbered/todo и callout note/warning/info в меню не разворачиваются — выбираются позже внутри блока).
3. **[info/console] 404 на seed-изображениях**: `/uploads/articles/event-loop.png` (ридер и review event-loop) — файла нет на диске, реальная загрузка изображений — Фаза 12. Даёт console error «Failed to load resource: 404» на каждой странице с этой картинкой — в спеках с проверкой консоли нужно вайтлистить.
4. **[minor] Гидрация /login**: до окончания гидрации клики по «Войти»/Enter молча игнорируются (POST не уходит, ошибок нет) — риск флаки-тестов; нужен ретрай/ожидание навигации.
5. **[info] Rate-limit 1/сек на PATCH** author-мутаций — быстрые последовательные API-вызовы в тестах получат 429.
6. **[info] Dirty-индикатор не сбрасывается** при возврате поля к исходному значению («не сохранено» остаётся, «Сохранить» активна) — ложноположительный сигнал изменений.
