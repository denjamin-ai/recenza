# Review-flow — доменные инварианты Recenza

Глава-ориентированная модель: **Blog → Chapter → Revision → blocks**. Ревью идёт по главам/ревизиям.

## Назначение и ведущий
- У главы — набор ревьюеров + **ведущий (primary)**. Вердикты хранятся **на handle и на ревизию**.
- `reviewer_history` фиксирует кредит по версиям — при новой ревизии прошлый кредит не перетирается.

## Согласие (модель назначения — обязательна)
- Ревьюер **НЕ активен по факту назначения**. Автор шлёт **приглашение** (`review_invitations`,
  статус `pending`). Ревью по нему стартует **только после `accepted`**. Автор узнаёт об ответе сразу.
- Ревьюер может пожаловаться **«навыки не совпадают»** (статус `flagged`) — допускается при match < 50%.
  Это **снимает главу с ревью**; автору предлагается исправить навыки (регуляция через админа).

## Вердикты и публикация
- Вердикт ревьюера: `approve | request-changes` (только в статусе ревизии `under-review`).
- Публикация главы автору доступна **только при всех `approve`** (или force-approve админом с уведомлением автора).
- Опубликованная глава указывает ревьюеров текущей версии + прошлых (за раскрытием).

## Apply-and-close (правки)
- Принятие suggestion: `chapter.blocks[i].text` ← replace(suggestion) в новой ревизии; тред → `resolved`.
  Снапшот предыдущей ревизии — ДО записи. Бродкаст статуса во все экраны (инбокс/кабинет/ридер).

## Подбор и «Топ» (фаза 9)
- **Компетенции ревьюера** (`users.competencies`) ≠ **навыки статьи** (`chapters.skills`). Навыки статьи —
  обязательны для отправки и видны читателю (отдельно от `blog.tags`).
- `match.pct` = доля навыков статьи, покрытых компетенциями. «Топ» = match 50% + рейтинг 30% + объём 20%.
- Занятость: `review_load/review_capacity` → `free | busy | full`; `full` не выбирается.

## Оценка (приватная)
- После публикации автор оценивает ревьюеров 1–5 (`reviewer_ratings`). **Приватно**: видят только ревьюер и
  админ. В «Топ» идёт лишь агрегат, отдельные оценки не раскрываются.

## Нет совпадений → админ
- Если матча нет — автор шлёт `recruit_requests` (вердикт: на рассмотрении/одобрен/отклонён+причина).
  Одобрение публикует направление на доску `board_calls`. **Блог нельзя опубликовать без ревью** — это запасной путь.

## Чеклист
- [ ] Ревью стартует только после accept приглашения; flag снимает главу с ревью
- [ ] Вердикты на handle/ревизию; публикация только при всех approve (или force-approve)
- [ ] Снапшот ревизии до записи; кредит по версиям не теряется
- [ ] Оценки приватны, в «Топ» — агрегат; занятость учитывается, full не выбирается
- [ ] Навыки статьи обязательны и видны читателю; recruit-запрос при отсутствии матча
