# DESIGN-TOKENS.md — Дизайн-токены Recenza (источник правды)

Единственный авторитетный источник стилей для миграции. Точные значения извлечены из рабочего
прототипа (`<style>` в `index.html`, блоки `:root` и `html[data-theme="dark"]`). **Фаза 1 (§1.5)
переносит эти токены в `src/app/globals.css` — значение в значение.** Любой UI-код ссылается
только на эти CSS-переменные (через Tailwind-утилиты вида `bg-[var(--accent)]`); **hardcoded-цветов
быть не должно** — это проверяет сабагент `design-watcher` в Цикле качества.

Эстетика (короткой строкой): спокойная редакторская — **serif-заголовки Lora**, **serif-текст
Literata**, **моно Fira Code**, **бирюзовый (teal) акцент**, **тонкие границы вместо теней**,
светлая/тёмная темы.

## 3. Скругления, тени, лэйаут

**Радиусы**

| Токен | Значение | Применение |
|---|---|---|
| `--radius-sm` | `0.25rem` | мелкие элементы |
| `--radius-md` | `0.375rem` | кнопки, чипы |
| `--radius-lg` | `0.5rem` | карточки, инпуты, блоки кода |
| `--radius-pill` | `9999px` | теги, пилюли сложности, статусы |

**Тени** — используются **крайне скупо**. Система предпочитает hairline-границы (`--border`)
drop-shadow’ам.

| Токен | Значение | Применение |
|---|---|---|
| `--shadow-focus` | `0 0 0 2px var(--accent)` | фокус-кольцо |

> Модалки используют локальный `box-shadow: 0 16px 40px rgba(0,0,0,0.2)` инлайном — это
> единственное исключение; в остальном плоско, на границах.

**Лэйаут-рельсы**

| Токен | Значение | Применение |
|---|---|---|

## 4. Движение

- Анимации только на `transform` / `opacity` (без layout-триггеров).
- Обязательная ветка `@media (prefers-reduced-motion: reduce)` — отключает анимации.
- Паттерн входа `.animate-in` — stagger fadeInUp через CSS-переменную `--index` (0-based).

---

## 5. Хит-таргеты и доступность

- Минимальный тач-таргет: **44×44px** (мобильные), **36px** допустимо для плотных десктоп-тулбаров.
- Видимое фокус-кольцо (`--shadow-focus` / `focus-visible:ring-2 ring-[var(--accent)]`) на всех интерактивных элементах.
- Контраст текста/фона — AA; `aria-live` для тостов; `role="tablist"` для strip’ов глав.

---

## 6. Правила использования (binding)

1. **Только переменные.** Никаких `text-red-500`, `#hex` в компонентах — лишь `var(--token)`
   (через Tailwind `bg-[var(--accent)]`, `text-[var(--muted-foreground)]` и т.п.).
2. **Заголовки — `--font-display`**, body — `--font-sans`, технические поля — `--font-mono`.
3. **Границы, не тени.** Разделение поверхностей — `--border`/`--border-secondary`, не shadow.
4. **Акцент — единственный «яркий» цвет.** Семантические — только для статусов (успех/ошибка/
   предупреждение/инфо), pin/amber — только для пин/портфолио.
5. **Тёмная тема обязательна** для каждого нового экрана; проверяется `design-watcher`.
6. **Tailwind-конфиг** маппит `fontFamily.{sans,display,mono}` на соответствующие переменные —
   держать в синхроне с этим файлом.

> При изменении любого токена правь **здесь**, затем `globals.css`, затем — синхронизируй
> Tailwind-конфиг. Этот файл — первоисточник.
